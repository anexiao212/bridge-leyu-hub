# bridge-leyu-hub

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不绑定任何特定域名或具体网站，仅作为静态页面的集中存储与展示平台。所有页面均为独立构建，可单独访问或部署。

## 目录说明

```
.
├── pages/           # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/          # 页面引用的公共资源（样式、图片、脚本等）
├── index.html       # 仓库首页（可选）
└── README.md        # 本文件
```

- `pages/`：核心目录，每个 HTML 文件代表一个独立页面，彼此无依赖关系。
- `assets/`：存放页面共享的 CSS、JS、图片等素材，便于统一管理。
- `index.html`：可选的仓库入口页面，用于导航或展示说明。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务。
- 页面之间相互独立，无跨页面引用或数据共享。
- 建议每个页面在文件名或页面标题中清晰标注其用途或主题，以便检索。
- 页面更新或新增时，请保持目录结构清晰，避免文件冲突。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面。
- 不建议直接修改他人已归档的页面文件，如有调整需求，请提交 Issue 或 Pull Request 讨论。
- 页面内容应遵守相关法律法规及社区规范，不得包含违法、侵权或不良信息。
- 如需删除或废弃某个页面，请将其移入 `archived/` 目录（如有）或直接移除，并更新索引。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/bridge-leyu-hub.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件即可预览。
3. 也可将整个仓库部署到任何静态托管服务（如 GitHub Pages、Netlify 等），实现线上访问。

## 贡献指南

欢迎提交 Issue 或 Pull Request 来优化仓库结构、修复页面问题或增加新页面。提交前请确保：
- 页面文件无外部依赖（或已在 `assets/` 中妥善托管）。
- 页面编码为 UTF-8，无乱码问题。
- 新增页面请在 `index.html` 中添加相应链接（如存在）。
